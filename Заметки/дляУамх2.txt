Идея: разработать уникальный идентификатор для объектов для моих проектов Хранилищ.
Назвать WIDE (World ID environment|enhancement)
Этот номер выдавать на сайте в сети, по запросу через АПИ, где нужно указать класс объекта и другие параметры.
- пока написать сервис виндовс, который будет эти идентификаторы выдавать моим приложениям, 
   которые к нему должны будут обращаться как к серверу

Цель: создать механизм генерации уникального идентификатора объектов для моих Хранилищ.
Задачи:
  - эту тему надо разрабатывать.

идеи:
- 256 бит длиной
- может содержать таймштамп создания
- создается строго уникально, внутри критической секции.
- содержит код класса объекта,
- содержит поле контрольной суммы, чтобы обнаружить ошибки при вводе или подбор номера.
- содержит 64-битное поле порядкового номера, которое инкрементируется независимо от   класса объекта.
  - подобно ид в тапп, но не обрабатывает удаление объекта, только создание.
- ид конвертируется в локальную веб-ссылку и обратно в ид, 
   чтобы пользователь мог кликать по ссылке в качестве указания объекта по его ИД.
